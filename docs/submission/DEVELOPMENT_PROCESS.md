# Development Process and Team Direction

This project used AI tools as an engineering copilot during the hackathon. The
team directed the product concept, sponsor strategy, demo priorities, testnet
operations, and final submission scope.

## Human-Led Decisions

The following decisions were made by the team and used to steer implementation:

- Product concept: a secondary market for idle AI capacity where agents buy and
  sell task execution instead of selling traffic or trusting sellers up front.
- Project scope: focus on agent-to-agent commerce, not a verifiable compute
  framework.
- Sponsor selection: remove 0G and ENS from the final scope; focus on Gensyn
  AXL, KeeperHub, and Uniswap.
- Trust model: use canonical content hashes, buyer verification, and escrow
  state transitions instead of claiming TEE or 0G proofs.
- Demo posture: be explicit about what is live/testnet, what is local replay,
  and what is a service-backed presentation screen.
- Testnet operations: create and fund wallets, deploy contracts, approve token
  spending, configure KeeperHub workflows, and validate lock/release/refund
  behavior.

## AI-Assisted Work

AI assistance was used for:

- Code review and implementation support.
- Removing unused 0G/TEE scaffolding.
- Writing and refining Python clients, scripts, tests, and documentation.
- Debugging Base Sepolia, KeeperHub, Uniswap, and Gensyn AXL integration issues.
- Drafting submission copy, demo flow, and sponsor feedback files.

All AI-assisted changes were reviewed, tested, and adjusted in the repository
before submission.

## Prompt and Direction Summary

The team repeatedly redirected the implementation during the hackathon:

- First, remove misleading 0G/TEE code and documentation.
- Then, deploy and test Base Sepolia escrow and MockUSDC settlement.
- Then, configure KeeperHub workflows for lock, release, and refund.
- Then, add Uniswap API quote/check proof without claiming a fake swap.
- Then, make Gensyn AXL a real integration using separate AXL nodes.
- Finally, prepare a demo screen and submission materials that clearly explain
  the market, matching, verification, and settlement flow.

This summary is included so judges can see how AI was directed by the team
rather than treating the final repository as an unguided generated artifact.
